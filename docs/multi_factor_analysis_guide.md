# 多因子量化策略分析系统

## 概述

多因子量化策略分析系统是基于现有单因子分析框架扩展的高级量化分析工具。该系统允许用户选择多个因子，手动设置权重，构建复合因子模型，并基于复合因子进行股票选择和回测分析。

## 主要特性

### 1. 多因子组合
- ✅ **灵活因子选择**：从6个预定义因子中选择任意组合
- ✅ **自定义权重**：为每个因子设置0.1-3.0的权重值
- ✅ **因子标准化**：可选择是否对因子值进行标准化处理
- ✅ **复合因子计算**：基于权重自动生成复合因子

### 2. 可用因子类型
1. **市值因子**：基于总市值或流通市值的小盘股策略
2. **动量因子**：基于价格动量的趋势跟踪策略
3. **价值因子**：基于市盈率(PE)的价值投资策略
4. **ROE因子**：基于净资产收益率的盈利能力策略
5. **波动率因子**：基于价格波动率的低波动策略
6. **换手率因子**：基于成交活跃度的流动性策略

### 3. 高级分析功能
- ✅ **因子相关性分析**：展示因子间的相关性热力图
- ✅ **权重影响分析**：可视化各因子权重分布
- ✅ **回测性能评估**：完整的收益率、夏普比率、最大回撤等指标
- ✅ **策略收益曲线**：对比基准指数的收益表现

## 使用方法

### 1. 启动系统
```bash
# 在项目根目录下运行
streamlit run backtesting/multi_factor_analysis.py --server.address 127.0.0.1 --server.port 6007
```

### 2. 配置策略参数

#### 因子选择与权重设置
1. 在左侧边栏中勾选要使用的因子
2. 为每个选中的因子设置权重值（0.1-3.0）
3. 系统会自动显示权重分布和百分比

#### 因子特定参数
- **市值因子**：
  - 市值计算方式：总市值/流通市值
  - 平滑窗口：1-20个交易日
- **动量因子**：
  - 动量回看期：5-60个交易日
- **波动率因子**：
  - 波动率计算周期：5-60个交易日
- **换手率因子**：
  - 换手率平滑周期：5-60个交易日

#### 回测参数
- **板块选择**：上证50、科创50、沪深300、中证500、中证1000、创业板
- **回测时间**：设置开始和结束日期
- **调仓周期**：5-60个交易日
- **持仓数量**：5-30只股票
- **因子标准化**：是否对因子值进行标准化

#### 前置过滤
- **市值范围过滤**：设置最小/最大市值（亿元）
- **上市时长过滤（自然日）**：优先使用基础信息`listed_date`，缺失时回退首个交易日
- 过滤在每个调仓日候选池中执行，便于后续持续扩展过滤条件

#### 风险控制
- **止损设置**：可设置1%-20%的止损比例
- **止损类型**：固定止损或移动止损

### 3. 运行回测
1. 点击"🚀 开始多因子回测"按钮
2. 系统将自动：
   - 加载股票数据
   - 计算各个因子值
   - 生成复合因子
   - 执行回测策略
   - 展示结果分析

### 4. 盘后选股与次日下单清单

盘后选股页新增了资金分摊下单能力，用于生成可直接给 QMT 使用的 `order.csv`：

- **总资金(元)**：默认 240000，可自定义
- **购买股票数量**：默认 12，可自定义
- **展示候选数量**：仅用于展示候选池，实际下单取前 N 只（N=购买股票数量）

数量计算规则：

1. 单票预算 = 总资金 / 购买股票数量
2. 股票价格来自 `xtdata.get_full_tick`，优先 `askPrice1`，缺失时回退 `lastPrice`
3. 下单数量 = 向下取整到 100 股整数倍
4. 无法获取实价或预算不足 100 股时，该股票不写入订单
5. 剩余资金保留，不做二次分配

页面会同时展示：实时价、价格来源、单票预算、下单数量、预计下单金额和备注。

## 系统架构

### 核心类说明

#### MultiFactorCalculator
多因子计算器主类，负责：
- 管理多个单因子计算器
- 处理因子权重设置
- 执行因子标准化
- 计算复合因子值
- 分析因子相关性

#### 关键方法
```python
# 添加因子
multi_factor_calc.add_factor("因子名", factor_calculator, weight)

# 计算复合因子
df_result = multi_factor_calc.calculate(df, factor_params)

# 分析相关性
corr_matrix = multi_factor_calc.analyze_factor_correlation(df, date)
```

### 算法流程

1. **因子计算**：对每个选中的因子分别计算因子值
2. **标准化处理**：按交易日对因子值进行Z-score标准化（可选）
3. **权重调整**：根据因子排序方向调整权重符号
4. **复合因子合成**：加权求和生成最终复合因子
5. **股票排序**：按复合因子值降序排列选择股票

### 复合因子计算公式

```
复合因子 = Σ(调整权重ᵢ × 标准化因子值ᵢ)

其中：
- 对于升序因子（小值更好），调整权重 = -原权重
- 对于降序因子（大值更好），调整权重 = +原权重
- 最终复合因子值越大，股票排名越靠前
```

### 交易统计

- **佣金成本**：交易手续费统计
- **换手率**：策略调仓频率

## 注意事项

1. **数据要求**：需要包含完整的价格、成交量、财务指标数据
2. **计算时间**：多因子计算比单因子更耗时，建议合理设置回测区间
3. **因子选择**：建议选择相关性较低的因子组合，避免信息重叠
4. **权重设置**：权重分配要平衡，避免某一因子权重过大
5. **参数调优**：不同市场环境下最优参数可能不同，需要动态调整

## 交易时序说明

1. 调仓信号在 t 日基于当日可见数据生成，不读取 t+1 数据。
2. Backtrader 在下一根 K 线执行订单（通常是 t+1）。
3. 若执行日停牌或不可交易，则该笔买卖跳过且不补单。
